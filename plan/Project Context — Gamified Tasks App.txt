# 📱 Project Context — Gamified Tasks App

## 🧠 Project Overview

Mobile application focused on **task management for families, couples or small groups**, introducing **gamification mechanics** to increase motivation and engagement.

The goal is to transform daily responsibilities into a **fun, reward-driven experience**, similar to habit-building systems used in modern productivity apps.

Users complete tasks → earn points → maintain streaks → level up → redeem rewards defined by group administrators.

Future vision includes evolving the platform into a **SaaS product**.

---

## 🎯 Core Idea

A user can:

* Create an account
* Create or join a group
* Collaborate with other members
* Complete assigned tasks
* Earn points and experience
* Maintain activity streaks
* Redeem rewards created by group owners

The system introduces gaming mechanics such as:

* Points
* Levels
* Streaks
* Rewards
* Progression

---

## 👥 User Roles

### Owner

* Creates group
* Invites members
* Creates tasks
* Assigns tasks
* Defines point values
* Validates completed tasks
* Creates rewards
* Approves reward redemption

### Sub-Owner

* Same operational permissions as Owner except ownership transfer

### Member

* Receives assigned tasks
* Marks tasks as completed
* Optionally uploads proof (image/text)
* Requests reward redemption

---

## 🧩 Group Model Decision

✅ Users **can belong to multiple groups**.

Examples:

* Family group
* Couple group
* Roommates
* School/team group

Limits will later be controlled via SaaS plans, not database structure.

---

## 🎮 Gamification Philosophy

The system rewards **events**, not tasks directly.

Example events:

* TaskApproved
* DailyActive
* RewardRedeemed
* StreakMaintained

Gamification systems listen to events and generate:

* Points
* XP
* Level progression
* Streak updates

This prevents future architectural rewrites.

---

## 🧱 System Separation

The app is divided conceptually into independent systems:

### 1. Task System

Handles:

* Tasks
* Assignment
* Completion
* Approval workflow

### 2. Points System

Responds to events and assigns points.

### 3. Progression System

Handles:

* XP
* Levels
* User growth

### 4. Reward System

Handles:

* Rewards catalog
* Redemption requests
* Approval flow

## 🚨 Important Development Rules

* Build functionality before gamification.
* Avoid implementing all systems simultaneously.
* Each stage must be stable before moving forward.
* Gamification reacts to events, never directly to UI actions.
